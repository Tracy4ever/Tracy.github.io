网页项目实时可视化

发表于 2017-08-26

本文主要总结的是如何更改文件，保存后，在网页实时看见我们的更改内容。
备注：系统windows7 64位 已安装nodeJS

主要用到三个工具：livestyle、livereload、browser-sync。

http服务器

要实现实时可视化更新，要先把网站运行在http服务器上。我用的是nodeJS的http-server，前提是得先安装nodeJS。在命令提示符中进入项目根目录，然后输入

就会提示本地服务器地址localhost：8080，在浏览器输入地址就能查看网站。（改文件名为index.html就会默认设置该文件为网站首页）

livestyle

这个工具主要用于检测css文件的修改并实时刷新页面(前面http-server后打开的页面)。
需要安装chrome浏览器插件和编辑器插件，我用的是sublime text3，利用package control 安装的livestyle。
sublime打开当前项目的html文件，点击启用livestyle的浏览器插件，更改css内容后ctrl+S即可实时刷新页面。

livereload

安装chrome浏览器插件和nodeJS插件:livereload.
全局安装livereload（注意，npm是nodeJS的模块管理器，需先安装nodeJS）

在本地项目中生成package.json文件，指明项目在开发环境和生产环境中的node模块依懒关系。

首先全局安装gulp

browser-sync

和livereload相似之处：

1.对所有的文件都可以实时更新。
2.也都是单向的，只能通过文件修改页面不能通过页面修改文件，
3.都支持chrome脚本

区别：

1.browser sync只依赖nodejs插件，不依赖浏览器插件。
2.局部刷新。
3.可以实现手机浏览器或者pc浏览器，多个浏览器之间的同步。包括视图同步和交互的同步
